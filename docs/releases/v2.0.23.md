# Discorder v2.0.23 - Discord baglanti dogrulama hotfix

Tarih: 2026-06-13

v2.0.23, v2.0.22 uzerine gelen baglanti dogrulama ve tanilama hotfix surumudur. Amac, uygulama "baglanti acik" dediginde kullanicinin neyin acik oldugunu daha net anlamasi, Discord calisiyorsa yeniden baslatma ihtiyacinin gorunmesi ve WireSock profilinin gercek Discord executable yollarini da kapsamasidir.

## Kisa Ozet

- Discord kurulum klasorundeki `app-*\\Discord.exe` yollari WireSock `AllowedApps` profiline eklenir.
- Tarayici modu acikken desteklenen tarayicilar icin bilinen tam executable yollari da profile eklenir.
- Tam executable yolu bulunan uygulamalarda bare process name fallback'i profile eklenmez; fallback yalnizca yol bulunamazsa kullanilir.
- Trusted arac, kurucu ve profil dosyalari production'da local appdata yerine protected shared data root altinda tutulur.
- Baglanti sonrasina yeni `Dogrulaniyor` asamasi eklendi.
- UI artik sadece WireSock sureci acik diye "Discord uygulamasi bagli" iddiasi kurmaz.
- Discord zaten aciksa kullaniciya net sekilde "Discord'u yeniden baslatin" mesaji verilir.
- Tanilama `health.json` dosyasi WireSock calisiyor mu, Discord sureci var mi ve sonraki kullanici aksiyonu ne bilgisini tasir.
- Tanilama paketine giren log kopyalari fake private key, token, authorization ve benzeri alanlari redakte eder.

## Ne Degisti?

- `DiscordAppScope`, bare process name fallback'lerini korurken gercek Discord ve tarayici exe yollarini da ekler.
- `DiscordAppScope`, tam yol bulunduysa bare process name fallback'ini atlar; bu, spoof edilebilir process-name kapsamını daraltır.
- `DiscordTunnelController`, WireSock baslatildiktan sonra `Verifying` durumuna gecer ve Discord sureclerini kontrol eder.
- `MainWindow`, yeni dogrulama durumunu progress bar, durum karti ve kullanici metinlerinde gosterir.
- Tanilama butonu controller'dan canli baglanti detaylarini alir; bu yuzden kullanici rapor hazirladiginda son baglanti kanitlari kaybolmaz.
- `DiscorderDiagnostics`, bundle icindeki loglari ham kopyalamak yerine redakte edilmis kopya olarak paketler.
- `AppPaths`, production'da `tools`, `profiles` ve `installers` klasorlerini shared data root altinda hazirlar; klasorler reparse/junction kontrolunden gecer.

## Neden Onemli?

Onceki surumlerde Discorder, WireSock sureci baslayip kisa sure icinde kapanmazsa baglantiyi basarili sayiyordu. Bu, tünel surecinin acik oldugunu gosterir; ancak Discord uygulamasinin var olan oturumunu toparladigini kanitlamaz.

Bu surum iki seyi duzeltir:

1. WireSock profili gercek Discord exe yolunu daha iyi yakalar.
2. UI kullaniciya "Discord aciksa yeniden baslat" adimini net soyler.

1. GitHub Releases sayfasindan `Discorder-2.0.23-win-x64.zip` arsivini indirin.

3. ZIP'i sabit bir klasore cikarin.
4. `Discorder.exe` dosyasini calistirin.

Uygulama ici guncelleme kullaniliyorsa once `Guncelle` ile yeni surum denetlenir. `Yukle` dugmesi gorundugunde indirilen paket GitHub release asset bilgisi, SHA-256 dosyasi ve update manifestiyle eslesmeden uygulanmaz.

- Paket once korumali staging alanina indirilir.
- ZIP ozeti, `.sha256.txt` dosyasi ve update manifest dogrulanmadan dosyalar uygulanmaz.
- Update helper dogru `Discorder.exe` dosyasini yeniden baslatir.
- Indirme, dogrulama veya staging hatasinda mevcut calisan klasor hedeflenmez.

## Guvenlik ve Guven Sinirlari

- WireSock kurucusu Authenticode, yayinci, surum ve SHA-256 kontrollerinden gecer.
- `wgcf` araci SHA-256 ile dogrulanir ve boyut siniriyla indirilir.
- Uygulama guncelleme paketi GitHub release asset digest, SHA-256 ve manifest kontrollerinden gecer.
- Tanilama paketindeki log kopyalari private key, token ve authorization benzeri alanlar icin redakte edilir.
- Trusted `wgcf.exe`, profil ve installer staging dosyalari production'da protected shared data root altinda tutulur.
- Bu build Authenticode imzali degildir; Windows SmartScreen uyarisi gorulebilir.

## Bilinen Sinirlar

- Kod imzalama sertifikasi olmadigi icin Windows yayinci adini isletim sistemi tarafindan dogrulanmis gostermeyebilir.
- GitHub erisimi olmayan aglarda guncelleme denetimi basarisiz olabilir; mevcut surum calismaya devam eder.
- Discord zaten aciksa yeni tünel kuralindan yararlanmasi icin tamamen kapatip yeniden acmak gerekebilir.
- Tarayici modu acikken desteklenen tarayicilarin genel uygulama sureci kapsama girer; bu yalnizca Discord sekmesini izole eden bir tarayici eklentisi degildir.
