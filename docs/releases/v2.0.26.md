# Discorder v2.0.26 - Discord acilis dogrulamasi

Tarih: 2026-06-13

v2.0.26, v2.0.25 sonrasinda gelen gercek kullanici tanilama paketindeki "bagli gorunuyor ama Discord acilmiyor" durumunu hedefler. Bu surumde Discorder artik Discord process'inin baslamasini tek basina yeterli saymaz; Discord ana penceresinin gorunur hale gelmesini de kisa sure dogrular.

## Kisa Ozet

- Discord kapaliyken baglanma akisi artik Discord'u otomatik acmayi dener.
- Discord Squirrel kurulumlarinda dogrudan `app-*\\Discord.exe` yerine `Update.exe --processStart Discord.exe` yolu tercih edilir.
- Restart veya acilis sonrasi gorunur Discord ana penceresi bulunamazsa "Baglanti hazir" denmez.
- Baglanti kapatilirken guvenilir Discord surecleri de kapatilmaya calisilir.
- Kullaniciya sakin ve net sekilde "Discord acildi ama pencere gorunmedi. Gorev cubugundan Discord'u acin." mesaji gosterilir.
- Tanilama detaylari restart denemesinden sonra tekrar process snapshot'i alir.

## Ne Degisti?

- `WindowsDiscordProcessInspector`, Discord'u baslatirken Squirrel `Update.exe` baslaticisini kullanacak sekilde guclendirildi.
- Discord hic acik degilse, normal Discord kurulumu `%LOCALAPPDATA%\\Discord\\Update.exe` uzerinden bulunup acilir.
- Sistem birden fazla Discord varyanti iceriyorsa hepsini acmak yerine oncelik normal Discord'a verilir.
- Baslatma sonrasinda `MainWindowHandle` ve gorunur pencere kontrolu yapilir.
- "Discord Updater" penceresi ana Discord penceresi gibi kabul edilmez.
- Minimize edilmis pencere gorunur sayilmaz; once geri yuklenir, sonra tekrar dogrulanir.
- `DiscordTunnelController`, Discord kapaliyken de manager'a acilis sorumlulugu verir.
- Restart/acilis basarisizsa UI ve diagnostic artik generic "hazir" yerine gercek kullanici aksiyonunu yazar.

## Neden Onemli?

Onceki surumlerde WireSock process'i calisiyor ve Discord process'leri gorunuyorsa uygulama baglantiyi hazir sayabiliyordu. Gercek kullanici deneyiminde Discord ana penceresi acilmadiysa bu yanilticiydi. v2.0.26 bu farki kapatir: tunel aktif kalabilir, ama Discord penceresi gorunmuyorsa kullaniciya bunun acikca soylenmesi gerekir.

## Kurulum ve Guncelleme Notlari

1. Uygulama icinden `Guncelle` butonuna basin.
2. Yeni surum bulunursa `Yukle` ile v2.0.26 paketini kurun.
3. Guncelleme tamamlaninca Discorder yeniden acilir.
4. Baglanirken Discord penceresi otomatik acilmazsa Discorder bunu ekranda ve tanilama paketinde acikca belirtir.

Manuel kurulumda GitHub Releases sayfasindan `Discorder-2.0.26-win-x64.zip` indirilir, sabit bir klasore cikarilir ve `Discorder.exe` calistirilir.

## Guvenlik ve Guven Sinirlari

- Discord baslatma yolu kullanicinin yerel Discord kurulumundan veya calisan Discord executable yolundan turetilir.
- Discord sureclerine yalnizca Authenticode dogrulamasi gecerli ve yayincisi `Discord Inc.` ise dokunulur.
- Reparse/junction iceren executable yollari guvenilir kabul edilmez.
- Discord, elevated Discorder alt sureci olarak degil Windows Shell uzerinden normal kullanici oturumu gibi acilir.
- Harici URL, release notu veya indirilen metin executable path olarak calistirilmaz.
- WireSock ve update paket dogrulama davranisi degismedi.
- Bu build Authenticode imzali degildir; Windows SmartScreen uyarisi gorulebilir.

## Bilinen Sinirlar

- Discord kendi hesabinda oturum, update veya tray davranisi nedeniyle pencereyi gostermeyebilir. Bu durumda Discorder artik bunu hazir baglanti gibi saklamaz.
- Discord dis servislerine erisim, Discord hesabinin durumu veya Discord'un kendi update/login akisindaki sorunlar Discorder tarafindan garanti edilemez.
- GitHub erisimi olmayan aglarda guncelleme denetimi basarisiz olabilir; mevcut surum calismaya devam eder.
