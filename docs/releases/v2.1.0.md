# Discorder v2.1.0 - Enterprise tanılama ve stabilizasyon

Tarih: 2026-06-14

v2.1.0, v2.0.x hotfix zincirini toparlayan ilk anlamlı minor sürümdür. Ürün davranışı geriye uyumludur: `BAĞLAN` dediğinizde Discorder yine yalnızca Discord bağlantısını yönetir; tarayıcı trafiği sadece Tarayıcı modu açıksa kapsama alınır. Bu sürümün farkı, bağlantı ve performans sorunlarını gelecekte tahminle değil kanıtla inceleyebilmek için opt-in debug tanılama katmanı eklemesidir.

## Kısa Özet

- Debug tanılama artık kullanıcı isterse açılır; varsayılan olarak kapalıdır.
- Debug açıksa tanılama paketine ayrı `debug.json` eklenir.
- `debug.json`, ağ adaptörü özetini, TCP durum sayılarını, CPU örneğini ve ilgili süreç RAM/handle/thread metriklerini içerir.
- Paket yakalama, komut satırı, token, cookie, private key, mesaj içeriği veya endpoint listesi toplanmaz.
- Private IP değerleri maskelenir; hassas anahtar isimleri değer içerikten bağımsız redakte edilir.
- Routing profil hash'i artık tam WireGuard profilini değil, yalnızca temizlenmiş routing satırlarını temsil eder.
- Tarayıcı modu değiştiğinde eski profil özetiyle karışan tanılama durumu temizlenir ve "bir sonraki bağlantıda uygulanacak" olarak raporlanır.
- Bağlantı hazırlık aşamaları süreleriyle loglanır: access lock, WireSock hazır, profil hazır, süreç başladı, Discord recovery gibi adımlar ayrılır.
- Release doğrulama script'i Gitleaks varsa secret scan'i kapıya ekler.

## Ne Değişti?

- `AppSettingsStore`, `DebugDiagnosticsEnabled` ayarını kalıcı ve geriye uyumlu şekilde saklar.
- Ana ekran Tanılama kartına `Debug tanılama` anahtarı eklendi.
- `DiscorderDiagnostics`, normal bundle'ı hafif tutar; debug açık değilse `debug.json` üretmez.
- Debug bundle; çalışma yolu, sürüm, OS, süreç metrikleri, ağ arayüzü özeti ve TCP state sayaçlarını secrets-safe biçimde yazar.
- `DiscordTunnelController`, mode değişiminde stale routing özetini kullanıcıyı yanıltmayacak şekilde sıfırlar.
- Connect akışına düşük maliyetli phase timing eklendi.
- Kalıcı log redaction kuralı anahtar adına duyarlı hale getirildi.
- Testler debug ayarı, bundle gating, redaction ve UI toggle görünürlüğünü kilitler.

## Neden Önemli?

Son kullanıcı için amaç değişmedi: Discorder açıkken genel internet yavaşlamamalı, Discord dışı trafik yönetilmemeli, tarayıcı dahil edilecekse bunu kullanıcı seçmeli.

v2.1.0 bu davranışı daha görünür ve ölçülebilir hale getirir. Kullanıcı "bağlıyken internet yavaş gibi" dediğinde normal tanılama hafif kalır; gerekirse Debug tanılama açılır ve ağ/performance resmi ayrıntılı ama secrets-safe şekilde alınır.

## Kurulum ve Güncelleme Notları

1. Uygulama içinden `Güncelle` butonuna basın.
2. Yeni sürüm bulunursa `Yükle` ile v2.1.0 paketini kurun.
3. Güncelleme tamamlanınca Discorder doğru `Discorder.exe` üzerinden yeniden açılır.

Manuel kurulumda GitHub Releases sayfasından `Discorder-win-x64.zip` indirilebilir. Sürümlü `Discorder-2.1.0-win-x64.zip` asset'i otomatik güncelleme doğrulaması ve release izlenebilirliği için korunur.

## Portable ZIP Davranışı

- Güncelleme mevcut çıkarılmış Discorder klasörünü hedefler.
- Paket önce korumalı staging alanına indirilir.
- ZIP özeti, `.sha256.txt` dosyası ve update manifest doğrulanmadan dosyalar uygulanmaz.
- Sürümlü ZIP adı release izlenebilirliği içindir; kullanıcı tarafında sabit `Discorder` klasörü önerilir.
- Eski sürümlü klasör adları tanılamada raporlanır; kullanıcı onayı olmadan klasör taşıma veya silme yapılmaz.

## Güvenlik ve Gizlilik

- Debug tanılama varsayılan olarak kapalıdır.
- Debug bundle komut satırı, packet capture, token, cookie, private key, Discord mesajı veya tam hassas config içermez.
- Private IP değerleri maskelenir.
- Public DNS adresleri, ağ adaptörü adları ve ilgili süreç metrikleri debug modda görünebilir; paylaşmadan önce paket içeriği gözden geçirilebilir.
- Bu build Authenticode imzalı değildir; Windows SmartScreen veya bazı AV uyarıları görülebilir.

## Bilinen Sınırlar

- Sertifika olmadığı için AV/SmartScreen itibarı tüm vendorlarda garanti edilemez.
- Tarayıcı modu açıksa WireSock kapsamı process seviyesindedir; browser içinde yalnızca Discord tab'ını ayırmak Windows/WireSock seviyesinde garanti edilemez.
- Debug tanılama daha ayrıntılıdır; normal kullanımda kapalı bırakılması önerilir.
