# Discorder v2.0.18 - İlk bağlantı hazırlığı daha net

Tarih: 2026-06-12

v2.0.18, gerçek tanılama paketlerinde görülen ilk bağlantı sorununu hedefler. WireSock kurulduktan sonra gerekli Cloudflare WARP aracı GitHub'dan indirilemediğinde uygulama sessiz kalabiliyor veya kullanıcı hangi aşamada olduğunu anlayamıyordu. Bu sürüm, o hazırlık adımını görünür, daha dayanıklı ve daha güvenli hale getirir.

## Kısa Özet

- İlk bağlantıda Cloudflare WARP aracı, hesap ve profil hazırlığı artık ana ekranda durum olarak görünür.
- Geçici DNS, bağlantı ve akış zaman aşımı hatalarında indirme daha fazla yeniden denenir.
- `wgcf` aracı SHA-256 doğrulaması korunarak ve boyut sınırıyla indirilir.
- Duran indirme zaman aşımı artık teknik hata yerine sade Türkçe mesajla gösterilir.
- Tarayıcı modu kapalıyken Discord masaüstü uygulaması kapsama alınır; desteklenen tarayıcılar profile eklenmez.

## Ne Değişti?

- `wgcf` indirme adımı artık canlı ilerleme bildirir.
- Cloudflare WARP hesap ve profil üretimi ayrı durum mesajlarıyla gösterilir.
- `wgcf` indirmesi için 32 MB üst sınır eklendi.
- Uygulama içindeki doğrulanmış indirme deneme sayısı kötü bağlantılar için 5'e çıkarıldı.
- Doğrudan stream idle timeout hataları kullanıcıya teknik metin olarak gösterilmez.
- Profil hazırlama progress akışı controller durumuna ve tanılama event'lerine bağlandı.

## Neden Önemli?

İlk kurulumda kullanıcı aslında iki ayrı hazırlık adımından geçer:

1. WireSock resmi kurucusu indirilir, doğrulanır ve Windows onayıyla kurulur.
2. Cloudflare WARP profili için `wgcf` aracı indirilir, doğrulanır ve Discord profili üretilir.

Önceki sürümde ikinci adımda GitHub DNS'i geçici düşerse kullanıcı sadece bağlantının başarısız olduğunu görüyordu. Artık uygulama hangi dosyanın hazırlandığını, indirme durumunu ve tekrar denemeyi daha açık gösterir.

## Kurulum ve Güncelleme Notları

1. GitHub Releases sayfasından `Discorder-2.0.18-win-x64.zip` arşivini indirin.
2. Yanındaki `Discorder-2.0.18-win-x64.sha256.txt` dosyasıyla arşivi kontrol edin.
3. ZIP'i sabit bir klasöre çıkarın.
4. `Discorder.exe` dosyasını çalıştırın.

Uygulama içi güncelleme kullanılıyorsa Discorder önce yeni sürümü denetler, uygun paket bulunduğunda `Yükle` adımını gösterir ve doğrulanan paketi mevcut portable klasör için hazırlar.

## Portable ZIP Davranışı

- Güncelleme mevcut çıkarılmış Discorder klasörünü hedefler.
- Paket önce kullanıcı veri klasöründe staging alanına indirilir.
- ZIP özeti, `.sha256` dosyası ve update manifest doğrulanmadan dosyalar uygulanmaz.
- Güncelleme helper süreci mevcut çalışan uygulamanın yerine geçişi yapar ve doğru `Discorder.exe` dosyasını yeniden başlatır.
- Bir indirme veya doğrulama hatası olursa mevcut çalışan klasör korunur.

## Güvenlik ve Güven Sınırları

- WireSock kurucusu Authenticode, yayıncı, sürüm ve SHA-256 kontrollerinden geçer.
- `wgcf` aracı SHA-256 ile doğrulanır ve boyut sınırıyla indirilir.
- Uygulama güncelleme paketi GitHub release asset digest, SHA-256 ve manifest kontrollerinden geçer.
- Bu build halen Authenticode imzalı değildir; Windows SmartScreen uyarısı görülebilir.

## Bilinen Sınırlar

- İlk çalıştırmada GitHub veya internet erişimi tamamen yoksa gerekli dosyalar indirilemez. Uygulama artık bunu daha açık gösterir ve tekrar deneme yapar.
- Tarayıcı modu kapalıyken tarayıcı süreçleri WireSock profiline eklenmez. Ek tarayıcı engeli Windows Firewall IP kapsamıyla uygulanır; tarayıcı içi cache, DoH veya DNS/IP değişkenliği nedeniyle bu katman mutlak web kill-switch olarak değerlendirilmemelidir.
- Sertifika olmadığı için Windows üzerinde yayıncı adı işletim sistemi tarafından doğrulanmış görünmez.
