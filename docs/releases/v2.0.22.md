# Discorder v2.0.22 - Premium UI ve tarayici modu hotfix

Tarih: 2026-06-13

v2.0.22, v2.0.21 uzerine gelen son UI ve kullanici deneyimi patch surumudur. Amac, animasyonlu arka plan videosunu ana ekranda geri getirmek, Tarayici modunu yeni kurulumlarda varsayilan acik yapmak ve ana kontrol yuzeyini daha premium, okunur ve hizli hissettirmektir.

## Kisa Ozet

- Arka plan videosu gorunur ana ekranda tekrar calisir.
- Minimize, tray ve kapatma yollarinda medya kaynagi yine serbest birakilir.
- Runtime CDN video fallback varsayilan kapali kalir; release paketi yerel `Assets/background.mp4` dosyasini kullanir.
- Tarayici modu yeni kurulumlarda varsayilan acik gelir.
- "Arka planda calis" metni "Arka planda calistir" olarak sadeleştirildi.
- Guncelle butonu daha parlak, daha belirgin ve premium gorunen ayri stile alindi.
- Ana progress bar daha kalin, renkli, glow ve parlama katmanli hale getirildi.
- Baglan/kes/kapanis metinleri guc ikonunun altina daha dengeli oturacak sekilde hizalandi.
- Sol ust uygulama ikonu ayni chip icinde daha buyuk ve gorunur yapildi.
- Surum bilgileri ve updater helper metadata `2.0.22` ile hizalandi.

## Ne Degisti?

- `MainWindow`, pencere gorunurken arka plan videosunu baslatir; pencere gizlenince veya minimize edilince `MediaElement` kaynagini kapatir.
- `DISCORDER_DISABLE_BACKGROUND_VIDEO=1` ile dusuk sistemlerde video tamamen kapatilabilir.
- `DISCORDER_BACKGROUND_VIDEO_REMOTE_FALLBACK=1` verilmedikce uygulama calisirken uzak video indirilmez.
- Fresh settings icin Tarayici modu default degeri acik yapildi. Kullanici kapatirsa bu tercih saklanir.
- Header copy, uygulama kapsam karti ve README/docs metinleri yeni Tarayici modu default'una gore guncellendi.
- Windows UI smoke testi, Tarayici modunun default acik geldigini ve video source davranisini dogrular.
- Core settings testi, legacy/missing settings dosyalarinda yeni Tarayici modu default'unu dogrular.

## Neden Onemli?

Kullanici ana ekrana baktiginda Discorder'in premium ve canli gorunmesi gerekiyor. v2.0.21'de performans icin bagli degilken video durdurulmustu; bu surum, gorunur ekranda animasyonu geri getirir ve performans korumasini yalnizca pencere gorunmezken devrede tutar.

Tarayici modu yeni kurulumda acik geldigi icin Discord Web kullanan kullanici ilk acilista ekstra ayar aramak zorunda kalmaz. Yalnizca masaustu uygulamasini kapsamak isteyen kullanici modu kapatabilir.

## Guvenlik ve Guven Sinirlari

- WireSock kurucusu Authenticode, yayinci, surum ve SHA-256 kontrollerinden gecer.
- `wgcf` araci SHA-256 ile dogrulanir ve boyut siniriyla indirilir.
- Uygulama guncelleme paketi GitHub release asset digest, SHA-256 ve manifest kontrollerinden gecer.
- Runtime arka plan videosu normal kullanimda GitHub/CDN'den medya cekmez; release paketi yerel `Assets/background.mp4` dosyasini icerir.
- Bu build Authenticode imzali degildir; Windows SmartScreen uyarisi gorulebilir.

## Bilinen Sinirlar

- Kod imzalama sertifikasi olmadigi icin Windows yayinci adini isletim sistemi tarafindan dogrulanmis gostermeyebilir.
- GitHub erisimi olmayan aglarda guncelleme denetimi basarisiz olabilir; mevcut surum calismaya devam eder.
- Yerel arka plan videosu bozuk veya eksikse uygulama statik arka planla devam eder. Runtime CDN fallback sadece `DISCORDER_BACKGROUND_VIDEO_REMOTE_FALLBACK=1` ile acilir.
- Tarayici modu kapatilirsa desteklenen tarayicilar WireSock profiline eklenmez. Ek web kapsami Windows Firewall katmaniyla yonetilir; tarayici cache'i, DoH veya DNS/IP degiskenligi nedeniyle bu katman mutlak web kill-switch olarak degerlendirilmemelidir.
