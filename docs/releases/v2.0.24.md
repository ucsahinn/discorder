# Discorder v2.0.24 - Discord yenileme ve tünel uyumluluğu hotfix

Tarih: 2026-06-13

v2.0.24, v2.0.23 sonrasında gerçek kullanıcı loglarında görülen "tünel açık ama Discord girmiyor" durumunu hedefler. Bu sürümde Discorder, WireSock süreci açık kaldı diye tek başına başarı varsaymaz; Discord zaten çalışıyorsa tünel açıldıktan sonra Discord'u güvenli şekilde yenilemeyi dener.

## Kısa Özet

- Discord zaten açıksa bağlantı sonrası otomatik yenileme denenir.
- Otomatik yenileme başarılıysa kullanıcı "Discord yenilendi. Bağlantı hazır" durumunu görür.
- Otomatik yenileme başarısızsa durum artık `Connected` değil, ayrı `DiscordRestartRequired` durumudur.
- Bu durumda kullanıcı net olarak "Discord'u yeniden başlatın" görür ve tünel hâlâ bağlantıyı kesilebilir aktif durumda kalır.
- WireSock `AllowedApps` uyumluluğu için bilinen Discord ve tarayıcı process adları geri eklendi.
- Tam executable path değerleri korunur; process adı fallback'i yalnızca bilinen uygulamalarla sınırlıdır.
- Otomatik Discord yenileme, PID tekrar kullanımına karşı canlı process adını ve executable yolunu yeniden doğrular.
- Stable, PTB, Canary veya Development birlikte açıksa her biri kendi executable yoluyla geri başlatılır.
- Canlı smoke script'i yeni ProgramData profil yolunu kontrol edecek şekilde güncellendi.

## Ne Değişti?

- `DiscordTunnelController`, çalışan Discord process'i görürse tünel açıldıktan sonra Discord'u yenilemeyi dener.
- `WindowsDiscordProcessInspector`, Discord process ID'lerini ve executable path'lerini yakalar.
- `WindowsDiscordProcessInspector`, kapatma öncesi canlı process'in hâlâ bilinen Discord süreci ve beklenen executable yolu olduğunu doğrular.
- `WindowsDiscordProcessInspector`, önce nazik kapanış, gerekirse process-tree kapatma ve ardından aynı Discord executable'larını yeniden başlatma akışını uygular.
- `TunnelState` içine `DiscordRestartRequired` eklendi.
- UI, manuel işlem gereken durumda artık "Bağlı" algısı oluşturmaz.
- Health ve diagnostics çıktıları `discordRestartStatus`, process sayısı, executable path sayısı ve sonraki aksiyonu birlikte yazar.
- `DiscordAppScope`, WireSock tarafında daha geniş uyumluluk için hem tam path hem de bilinen process adlarını profile yazar.

## Neden Önemli?

Discord açıkken başlatılan tünel mevcut Discord oturumuna her zaman uygulanmayabilir. Eski davranışta WireSock açık kaldığı için uygulama bağlantıyı başarılı sayabiliyordu. Bu sürüm, Discord oturumunu yenileyerek bağlantının yeni Discord sürecine uygulanmasını sağlar.

1. GitHub Releases sayfasından `Discorder-2.0.24-win-x64.zip` arşivini indirin.
2. Yanındaki `Discorder-2.0.24-win-x64.sha256.txt` dosyasıyla arşivi kontrol edin.
3. ZIP'i sabit bir klasöre çıkarın.
4. `Discorder.exe` dosyasını çalıştırın.

Uygulama içi güncellemede `Güncelle` önce yeni sürümü denetler. `Yükle` görüldüğünde indirilen paket GitHub release asset bilgisi, SHA-256 dosyası ve update manifestiyle eşleşmeden uygulanmaz.

## Güvenlik ve Güven Sınırları

- WireSock kurucusu Authenticode, yayıncı, sürüm ve SHA-256 kontrollerinden geçer.
- `wgcf` aracı SHA-256 ile doğrulanır ve boyut sınırıyla indirilir.
- Uygulama güncelleme paketi GitHub release asset digest, SHA-256 ve manifest kontrollerinden geçer.
- Discord otomatik yenileme, doğrulanamayan PID veya executable yolunu kapatmaz; kullanıcıdan manuel yeniden başlatma ister.
- `AllowedApps` process adı uyumluluğu yalnızca bilinen Discord ve desteklenen tarayıcı adlarıyla sınırlıdır; kullanıcıdan rastgele process adı alınmaz.
- Bu build Authenticode imzalı değildir; Windows SmartScreen uyarısı görülebilir.

## Bilinen Sınırlar

- Discord çok yüksek yetkiyle veya farklı kullanıcı oturumunda çalışıyorsa otomatik yenileme başarısız olabilir. Bu durumda uygulama açıkça Discord'u elle yeniden başlatmanızı ister.
- Tarayıcı modu açıkken desteklenen tarayıcının genel uygulama süreci kapsama girer; yalnızca tek Discord sekmesini izole eden bir tarayıcı eklentisi değildir.
- GitHub erişimi olmayan ağlarda güncelleme denetimi başarısız olabilir; mevcut sürüm çalışmaya devam eder.
