# Discorder v2.0.20 - Update log ve premium hareket polish

Tarih: 2026-06-13

v2.0.20, v2.0.19 uzerindeki premium arayuz katmanini koruyarak update sirasindaki gorunurlugu ve performansi toparlayan patch surumudur. Kullanici ilerlemeyi ekranda gormeye devam eder; tanilama loglari artik binlerce tekrar eden indirme satiriyla sismez.

## Kisa Ozet

- Otomatik guncelleme indirme ilerlemesi log dostu sekilde seyreltilir.
- Update ekrani kullaniciya anlamli ilerleme ve durum bilgisini gostermeyi surdurur.
- Alt komut butonlarinin uzerindeki sert duz cizgiler kaldirildi.
- Header logosuna hafif, transform tabanli premium mikro animasyon eklendi.
- Surum bilgileri ve updater helper metadata `2.0.20` ile hizalandi.

## Ne Degisti?

- `AppUpdateService`, cok sik gelen indirme callback'lerini yuzde bucket'lariyla sinirlar.
- `MainWindow`, ayni update mesajini tanilama loguna yalnizca anlamli yuzde araliklarinda yazar.
- `Theme.xaml` komut butonlarindaki ust highlight cizgilerini kaldirir; footer daha temiz gorunur.
- `MainWindow.xaml` logo hareketini `RenderTransform` uzerinden calistirir; layout yeniden olculmez.
- Core testlere, 5000 indirme callback'i gelse bile UI/update progress olaylarinin makul sayida kaldigini dogrulayan regresyon testi eklendi.

## Neden Onemli?

Kotu veya yavas internet kullanan kullanicilarda update sirasinda indirme callback'i cok sik tetiklenebiliyor. Onceki davranis kullaniciya ilerleme gostermekle birlikte `events.jsonl` dosyasini gereksiz buyutuyordu.

Bu surumde ilerleme gorunurlugu korunur, fakat log yazimi seyreltilir. Tanilama paketi daha okunabilir kalir ve update sirasinda gereksiz disk yazimi azalir.

## Kurulum ve Guncelleme Notlari

1. GitHub Releases sayfasindan `Discorder-2.0.20-win-x64.zip` arsivini indirin.
2. Yanindaki `Discorder-2.0.20-win-x64.sha256.txt` dosyasiyla arsivi kontrol edin.
3. ZIP'i sabit bir klasore cikarin.
4. `Discorder.exe` dosyasini calistirin.

Uygulama ici guncelleme kullaniliyorsa Discorder once yeni surumu denetler. `Yukle` dugmesi gorundugunde indirilen paket GitHub release asset bilgisi, SHA-256 dosyasi ve update manifestiyle eslesmeden uygulanmaz.

## Portable ZIP Davranisi

- Guncelleme mevcut cikarilmis Discorder klasorunu hedefler.
- Paket once korumali staging alanina indirilir.
- ZIP ozeti, `.sha256.txt` dosyasi ve update manifest dogrulanmadan dosyalar uygulanmaz.
- Update helper dogru `Discorder.exe` dosyasini yeniden baslatir.
- Indirme, dogrulama veya staging hatasinda mevcut calisan klasor hedeflenmez. Uygulama asamasinda sorun olursa updater geri yukleme dener; dosya kilidi, antivirus veya yetki problemi geri yuklemeyi de engellerse kullanici dogrulanmis ZIP'i tekrar cikarmalidir.

## Guvenlik ve Guven Sinirlari

- WireSock kurucusu Authenticode, yayinci, surum ve SHA-256 kontrollerinden gecer.
- `wgcf` araci SHA-256 ile dogrulanir ve boyut siniriyla indirilir.
- Uygulama guncelleme paketi GitHub release asset digest, SHA-256 ve manifest kontrollerinden gecer.
- Bu build Authenticode imzali degildir; Windows SmartScreen uyarisi gorulebilir.

## Bilinen Sinirlar

- Kod imzalama sertifikasi olmadigi icin Windows yayinci adini isletim sistemi tarafindan dogrulanmis gostermeyebilir.
- GitHub erisimi olmayan aglarda guncelleme denetimi basarisiz olabilir; mevcut surum calismaya devam eder.
- Tarayici modu kapaliyken desteklenen tarayicilar WireSock profiline eklenmez. Ek web kapsami Windows Firewall katmaniyla yonetilir; tarayici cache'i, DoH veya DNS/IP degiskenligi nedeniyle bu katman mutlak web kill-switch olarak degerlendirilmemelidir.
